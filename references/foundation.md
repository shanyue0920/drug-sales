# 内置共享知识库（本 skill 自带 · 源自 drug-base 共享底座）

> 本文件由药品分析体系共享底座(drug-base)的共享内容合并而来，随本 skill 一同发布。
> 目的：让本 skill **完全自包含、可独立运行**，无需安装 drug-base 或其他任何 skill。

## 一、临床档案提取框架
给定药品/治疗领域，提取以下基础临床档案（各角色 skill 按需取用，深度由各角色展开）：

1. **基本信息与定位**：通用名/商品名/企业/剂型/规格/批准文号/药理分类 + 一句话定位（临床价值主张 ∩ 商业定位）
2. **作用机制与靶点**：通路、靶点、药理分类（见本 skill `references/drug-classes.md`）
3. **适应症与循证证据**：按 `references/evidence-levels.md` §4 **强制罗列循证三件套**——
   - ①研究数据（关键 RCT/Meta/RWE：研究名·设计·样本量·关键终点·年份·证据等级）
   - ②指南推荐（指南名·机构·年份·推荐级别·具体地位）
   - ③专家共识（共识名·机构·年份·核心建议）
   - 任一缺失标 💭 待核实，禁止用单源/厂商材料替代独立循证
4. **用法用量与特殊人群**：剂量、途径、儿童/孕妇/肝肾注意
5. **药代动力学(PK)**：ADME、半衰期、达峰时间
6. **安全性**：不良反应/禁忌/相互作用/黑框警示
7. **注册审批与在研管线（基础）**：NMPA/FDA 状态、专利、在研（深度由注册角色 skill 展开）

## 二、数据校准规则（强制 · 所有输出必须遵守）
本体系输出的每条关键数据都必须可溯源、可校验。严格执行以下 5 条：
- **来源分级标注**：每条关键数据标可信度——🔍 已查证（官方说明书 / NMPA·FDA / 权威指南 / 核心文献）、💭 推测或不确定（模型推断或单一非权威来源）。禁止把 💭 当 🔍 呈现。
- **关键事实交叉校验**：批准文号、医保类别、集采状态、价格、获批适应症等，须至少 2 个独立来源一致方可写为确定；不一致或仅单源时标「待核实」。
- **时效校准**：集采/医保/审批/在研状态必须 WebSearch 取最新，并标注「截至 YYYY-MM」，不得沿用旧记忆。
- **口径校准**：市场规模/患者数注明统计口径（终端价/出厂价、中国/全球、确诊/估测）；估算项显式标「估算」。
- **禁止编造**：任何无法溯源的数据标「待核实」，不得生成看似合理但无来源的数字。

## 三、专利 FTO 基础框架
（合规/法务、商务 BD 角色重点引用）
- **专利状态核查**：化合物专利 / 晶型专利 / 制剂专利 / 用途专利的到期日与地域覆盖
- **FTO（自由实施）分析要点**：是否落入他人有效专利权利要求范围
- **专利悬崖与延长保护**：专利期补偿（PTE）、数据保护、市场独占期
- **数据校准**：专利号、到期日须以专利数据库（CNIPA / USPTO / 智慧芽等）为准，标 🔍；不得凭记忆给专利号

## 四、完整性规则（防偷懒 · 强制）
- 临床档案各维度查不到内容时保留空字段并标「待核实（💭）」，禁止删除维度或以「无/暂略」跳过。
- 禁止编造批准文号、临床数据、价格、获批状态填充空字段。
- 不存在 ≠ 可删：确无公开信息时写明「未检索到公开 {X} 信息（💭）」，而非不输出该维度。
- 结构顺序不可因内容多寡重排或省略。
